# Xây dựng blog tĩnh với TanStack Start

Trong bài viết này, tôi sẽ chia sẻ cách xây dựng blog mà bạn đang đọc — từ việc chọn công nghệ đến triển khai chi tiết.

## Tại sao TanStack Start?

TanStack Start là một full-stack React framework mới, hỗ trợ **static site generation (SSG)** rất tốt. So với Next.js hay Remix, nó có một số ưu điểm:

| Tính năng | TanStack Start | Next.js | Remix |
|-----------|---------------|---------|-------|
| Type-safe routing | ✅ Built-in | ❌ Cần thêm | ❌ Cần thêm |
| Static export | ✅ Nitro | ✅ Built-in | ❌ Hạn chế |
| Bundle size | Nhỏ | Lớn | Trung bình |
| File-based routing | ✅ | ✅ | ✅ |

## Cài đặt dự án

Đầu tiên, khởi tạo project và cài đặt dependencies:

```bash
mkdir my-blog && cd my-blog
npm init -y
npm install @tanstack/react-router @tanstack/react-start react react-dom
npm install -D tailwindcss @tailwindcss/vite @content-collections/core
```

## Cấu hình Vite

File `vite.config.ts` là trung tâm của dự án:

```typescript
import { defineConfig } from "vite";
import { contentCollections } from "@content-collections/vite";
import tailwindcss from "@tailwindcss/vite";
import { tanstackStart } from "@tanstack/react-start/plugin/vite";

export default defineConfig({
  plugins: [
    contentCollections(),
    tailwindcss(),
    tanstackStart({ target: "static" }),
  ],
});
```

<Callout type="tip" title="Mẹo">
Thứ tự plugin rất quan trọng! `contentCollections()` phải đặt trước `tanstackStart()` để đảm bảo content được xử lý trước khi build routes.
</Callout>

## Xử lý MDX với Content Collections

Content Collections giúp type-safe content management. Mỗi bài viết MDX có frontmatter được validate bằng Zod:

```typescript
const posts = defineCollection({
  name: "posts",
  directory: "content/posts",
  include: "**/*.mdx",
  schema: (z) => ({
    title: z.string(),
    date: z.string(),
    description: z.string(),
    tags: z.array(z.string()).optional(),
  }),
  transform: async (document, context) => {
    const mdx = await compileMDX(context, document, {
      remarkPlugins: [remarkGfm, remarkMath],
      rehypePlugins: [rehypeShiki, rehypeKatex],
    });
    return { ...document, mdx, slug: document._meta.path };
  },
});
```
